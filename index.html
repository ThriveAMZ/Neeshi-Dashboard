Methodology

Profit & Loss figures are pulled directly from Amazon's SKU Economics report, exported at the Child ASIN aggregation level so that cost of goods sold can be applied accurately per child ASIN. COGS per unit (configurable in the Files tab) is applied against net units sold (units kept after returns) to produce true profit. Returned units are not re-charged COGS as Amazon's report already accounts for refund admin fees and referral fee refunds in net proceeds. Sponsored Products charges in the SKU Economics file include all SP campaign spend across active and paused targets in the period.

The advertising chart and targeting heat map are built from Amazon's Targeting export. Spend shown there may differ slightly from the Sponsored Products charge in the P&L, which uses a different attribution window. The heat map compares the two periods chosen in the selectors at the top of the section: each row shows the selected metric in the "from" period, the change, and the "to" period. Click any of the three column headers to sort by that column, or click a target row to expand a full side-by-side comparison of every metric for that target across the two periods. The paused and dormant filters control which target statuses are shown. The change cell is colored green when a target moved in a favorable direction and red when it moved unfavorably (reversed for ACoS and CPC, where lower is better); spend and impressions, which have no inherent good or bad direction, use a blue/slate scale for increase and decrease. Color intensity scales with the size of the percent change.

Data management

Export downloads the current state as data.json. Push this to the client's GitHub repo to update their hosted dashboard.

Import loads a previously exported data.json back into this browser, so you can resume work on a client.

Clear wipes this browser's saved data and reloads. If the page is hosted with a data.json, that loads next; otherwise the page returns to its built-in empty state.

ASIN catalog
Set cost of goods sold per unit for each child ASIN. Because COGS varies between child ASINs, export the SKU Economics report at the Child ASIN aggregation level so each one gets its own accurate cost. This drives the profit lines in the P&L.
